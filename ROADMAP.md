# Roadmap Público del Proyecto 🗺️

Este documento describe la hoja de ruta estratégica para la **Guía de Programación con AI**. Aquí puedes ver las características que estamos planeando, las que están en desarrollo y lo que ya se ha lanzado.

Puedes proponer adiciones a este Roadmap abriendo un **Issue** con la etiqueta `proposal`.

---

## 🟢 Lanzado (Versión 3.0)
- [x] **Arquitectura Modular**: Separación de la guía en carpetas individuales por módulo para facilitar la lectura y contribución.
- [x] **Proyectos Ejecutables**: Código fuente real y funcional para los 5 proyectos prácticos de la guía.
- [x] **Configuración Web**: Integración con GitHub Pages y Jekyll para visualización web premium.
- [x] **GitHub Actions**: Automatización semanal para la actualización y mantenimiento de la tabla de modelos y herramientas.

---

## 🟡 En Desarrollo (Q3 2026)
- [ ] **Módulo de Agentes Multi-Agente**: Expandir la guía con casos prácticos usando **CrewAI** y **LangGraph** interactuando entre sí.
- [ ] **Tutoriales en Formato Short-Video**: Enlaces a demos rápidas de 1 minuto sobre cómo usar Cursor o Copilot.
- [ ] **Ejercicios Interactivos**: Configurar un playground básico o notebooks de Jupyter (`.ipynb`) en Google Colab para probar las APIs sin instalar nada localmente.

---

## 🔵 Próximamente (Q4 2026 / 2027)
- [ ] **Vibe Coding avanzado**: Guía paso a paso para construir aplicaciones web completas (Frontend + Backend) en pocas horas utilizando únicamente instrucciones en lenguaje natural.
- [ ] **Fine-Tuning Local**: Guía detallada para entrenar modelos pequeños (como Llama 3 o Phi 3) de manera local utilizando recursos accesibles.
- [ ] **RAG Avanzado (Agentic RAG)**: Arquitecturas donde los agentes deciden qué base de datos vectorial o herramienta de búsqueda consultar según la complejidad del prompt del usuario.
- [ ] **Traducción y localización**: Adaptación de términos del ecosistema para comunidades específicas de desarrollo en América Latina y España.

---

## 📊 ¿Cómo nos organizamos?
Usamos el tablero de **GitHub Projects** del repositorio para hacer el seguimiento detallado de las tareas del día a día (Bugs, Features, Refactors). Si quieres sumarte a una tarea en desarrollo, deja un comentario en el Issue correspondiente y te la asignaremos.
